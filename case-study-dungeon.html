Role & Team

My role: Graphics Engineer & Interaction Designer

Team: Collaborated with two classmates on C++/OpenGL rendering

Context & Problem

For “Dungeon Masters” we set out to build an immersive, procedurally-generated dungeon simulator.

  • Technical pipeline: Implement the OpenGL real-time rendering pipeline in C++ using normal mapping, Bézier-curve camera paths, and dynamic lighting.
  • Game design: Craft the visual basis for a video game or animation by randomly generating maze layouts and guiding the user’s viewpoint along smooth, cinematic camera curves.

Research

I took CS1230: Introduction to Graphics at Brown University. It offered an in-depth exploration of fundamental concepts in 2D and 3D computer graphics.

  • It introduces 2D raster graphics techniques (image creation/manipulation/filtering)
  • 3D modeling, viewing, and rendering (using both raytracing and real-time rendering on the GPU).
  • Learned to program in C++ and the shading language GLSL, and to use the OpenGL library.

Process & Iterations

Key steps in building our simulation:

  • Normal mapping: Layer textures to give walls and floors realistic texture.
  • Bezier camera: Program smooth camera movement using the Bezier algorithm.
  • Random generator: Used A* algorithm to create physical random connections.

Solution

The final demo:

  • A real-time OpenGL pipeline with customizable shaders.
  • Procedural dungeon layouts that can be changed by the user.
  • Smooth Bézier-curve camera paths that guide the user throughout the scene.

Learnings & Next Steps

  • A new interest in animation and game design.
  • A crash course on graphics and graphics computing.
  • Insight into design beyond web and branding. I got to consider what it might look like to design a world.
